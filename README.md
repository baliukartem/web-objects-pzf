# ПЗФ Вінниччини

Веб-застосунок для перегляду територій та обʼєктів **природно-заповідного фонду Вінницької області** станом на **01.01.2025**. Дані синхронізовано з офіційним переліком, привʼязано до **нової адміністративно-територіальної структури** (6 районів + 63 ОТГ) з використанням ID із наданих DBF/шейпфайлів.

## Швидкий старт

```bash
cd pzf-vinnytsia
python3 server.py
```

Браузер відкриється сам на `http://localhost:8000/`. Якщо ні — відкрийте вручну.

> ⚠️ Не відкривайте `index.html` подвійним кліком (через `file://`) — браузер заблокує завантаження локальних JSON-файлів. Завжди користуйтесь сервером.

## Що в інтерфейсі

- **Карта (Leaflet + CartoDB):** межі районів, межі громад (тогл-шар), маркери всіх обʼєктів ПЗФ, кольори за категорією.
- **Кластеризація** маркерів (вмикається/вимикається).
- **Сайдбар з фільтрами:** пошук, район, ОТГ (динамічно фільтрується за вибраним районом), категорія, тип, бейджі «Загальнодержавне / Місцеве».
- **Список** результатів зі сумарною площею.
- **Деталі обʼєкта:** тип, площа, район, ОТГ, землекористувач, підстава створення, координати з міткою джерела (OSM / в межах ОТГ / вручну).
- **Статистика** (модальне вікно): розподіл за категоріями, площа за районами, ТОП-10 типів.

## Координати обʼєктів — як вони зʼявляються

У офіційних джерелах **немає координат**, лише текстовий опис на кшталт «Бершадське лісництво кв. 54-55». Тому координати визначаються в три кроки:

1. **`build_data.py`** ставить кожен обʼєкт у детерміновану точку всередині полігону його ОТГ (різні обʼєкти однієї громади — різні точки, але всі в межах громади).
2. **`geocode_osm.py`** (опціонально) парсить `location` кожного обʼєкта, шукає згаданий населений пункт або лісництво в OSM, і підставляє реальні координати.
3. **`coordinates_override.json`** дає змогу руками задати точні координати для конкретних обʼєктів — вони мають найвищий пріоритет.

В UI кожна координата позначена джерелом:

- 🟢 **точна (вручну)** — з `coordinates_override.json`
- 🟢 **з OSM** — підставлено з OpenStreetMap після `geocode_osm.py`
- ⚪ **в межах ОТГ** — детермінована точка всередині полігону громади
- ⚪ **в межах району** — fallback, коли ОТГ не визначено

## Як отримати реальні координати з OSM

1. У Cowork mode додайте дозвіл мережі для домену `overpass-api.de` (Settings → Capabilities → Network).
2. Запустіть:

   ```bash
   pip install -r scripts/requirements.txt
   python3 scripts/geocode_osm.py
   ```

3. Скрипт завантажить усі населені пункти та лісництва Вінницької області з OSM, спробує знайти збіг для кожного обʼєкта ПЗФ і оновить `assets/data/pzf.json`.
4. Перезапустіть сервер та перезавантажте сторінку — маркери стануть на реальні місця.

Скрипт зберігає сирі дані OSM у `assets/data/osm_settlements.json` та `osm_forests.json`, тож повторні запуски не вимагають мережі.

## Як виправити окремі обʼєкти вручну

Відкрийте `assets/data/coordinates_override.json` і додайте записи у форматі:

```json
{
  "objects": {
    "«Кармелюкове Поділля»": [48.2200, 29.5500],
    "Бронницький": [48.4456, 27.7853],
    "Урочище \"Устянська дача\"": [48.7625, 29.3935]
  }
}
```

Ключ — назва обʼєкта точно як у БД (поле `name`). Значення — масив `[широта, довгота]` у WGS84. Після збереження просто оновіть сторінку браузера.

## Як перебудувати дані з нуля

Якщо ви оновите вихідні файли в `source/` (новий PDF, оновлені шейпи), запустіть:

```bash
pip install -r scripts/requirements.txt
python3 scripts/build_data.py
```

Скрипт повністю перегенерує `pzf.json`, `rayons.geojson`, `otgs.geojson` з вихідних даних. Файли в `source/` ніколи не модифікуються.

## Технічні деталі

- **Шейпфайли** прочитані pure-Python парсером (без GDAL), CRS — **WGS84** (EPSG:4326), безпосередньо придатні для вебу.
- **OTG.dbf** містить латинські символи-двійники (`a`, `e`, `o`, `p`) у назвах громад — `shp_parser.py` автоматично нормалізує їх у кирилицю.
- **PDF** парситься через `pdfplumber`. Розпізнано 421 з 434 обʼєктів офіційного переліку (97 %); решта 13 розкидані по сторінкових розривах і потребують ручної правки в `coordinates_override.json` або правки `pdf_parser.py`.
- **Привʼязка до ОТГ** через regex по полю `location`: спочатку шукається повна форма «<основа>(а|у|ого|ому|ою|ій) (сільська|селищна|міська) територіальна громада», далі — підрядкові співпадіння; пріоритет — кандидат у тому самому районі.
